Developer Mode

⚠️ Read-only console — only read services allowed. Write/actuate/clear services are blocked before the bus.
What is this? How do I use it?

This sends a raw diagnostic command to one module and shows the reply. You need two things:

1. Addr — which module to talk to (its bus address, in hex):

12DME (engine) 60IKE / cluster
56LCM (lights) 00GM (body/central)
5BIHKA (climate) D0DSC (ABS)

2. Payload — the service (command) to run, in hex. Read-only ones:

00Identify (part no, sw version)
04Read fault memory
0B 03Read status block 03 (live values)
21 5ARead data record 5A (e.g. battery V)

The frame wrapper (length byte + checksum) is added for you — type only the service bytes. Anything that writes, actuates, or clears is refused here on purpose.
Where to find more commands: the Protocol Explorer tab lists 400+ known jobs per module (with evidence grades), and docs/OPERATIONS.md documents them.

Quick fill:
Examples: 00 (ident), 04 (read faults), 0B 03 (status), 21 5A (read data)

RAM Explorer

Read-only live memory viewer (E39/MS41, service 0B 01/00). Capture RAM at idle vs. higher RPM, then Diff to reveal which addresses track a parameter — how you locate an unknown sensor's address.
What is this? How do I use it?

The DME keeps every live value (RPM, coolant, load…) somewhere in its RAM. This tool reads raw bytes at an address so you can find which byte holds which value — the groundwork for logging a signal that has no named parameter yet.

To hunt for an unknown value's address:

  1. At idle, press Capture A.
  2. Hold the engine at higher revs (or change the thing you're chasing), press Capture B.
  3. Press Diff A↔B. Bytes that changed are candidates; the one that tracks RPM is your address.

Where do the addresses come from? Start from a byte you already know and read a window around it. Known MS41 addresses (use the quick-pick below):

DA2ARPM (2 bytes) DA5ACoolant temp
DA50Intake air temp DA63Vehicle speed
FAFCEngine load E9E6VANOS

Full list lives in ms41_ram_params.json. This panel only reads — it never writes to the ECU.
